HANDOFF NOTES

Build spec for Claude Code

SCORING (max 76)
  • Correct group winner = 3
  • Correct group runner-up = 2
  • Each correct best-3rd group = 2 (8 to pick)
  • Per group max 5 · 12×5 = 60 · + 8×2 = 76
DATA MODEL
  • Player { id, name, createdAt }
  • Pick { playerId, group A–L: {first, second} }
  • ThirdPicks { playerId, groups[8] }
  • Results { group: {first, second}, best3rdGroups[8] }
  • Settings { entriesOpen, picksRevealed, adminCode }
HOSTING (Render)
  • Friends each open on their own phone → needs shared storage, not localStorage.
  • Tiny backend + DB (SQLite/Postgres). One web service on Render.
  • Admin guarded by a single passcode env var.
RULES & STATES
  • Identity = name only (no auth). Name stored in a cookie so they can re-edit.
  • Final on submit — no edits once a player locks in.
  • Picks hidden from friends until admin Reveals.
  • Re-score automatically whenever results change.
✓ DECISIONS — LOCKED
Best-3rd = GROUP level. Pick 8 of 12 groups whose 3rd-place team advances; 2 pts per correct group (no specific-team naming). · Picks final on submit — no edits afterwards. · Tie-breaker handled manually by the admin if it ever comes up (no in-app rule needed).
